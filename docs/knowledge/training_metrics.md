# YOLO 训练指标详解

## 1. mAP（Mean Average Precision，平均精度均值）

**最核心的检测模型评价指标。**

### mAP@50
- 含义：IoU阈值设为0.5时的mAP。只要预测框和真实框重叠超过50%，就算检测正确。
- 解读：**衡量"找没找到目标"的能力**。数值越高说明模型定位越准。
- 范围：0~1，越接近1越好。
- 当前值0.51：模型能正确检测约51%的目标（IoU≥0.5标准）。

### mAP@50-95
- 含义：从IoU=0.50到0.95，每0.05取一个阈值，计算所有阈值下的mAP平均值。
- 解读：**COCO标准指标，比mAP@50更严格**。要求框的位置非常精确才能得分。
- 范围：0~1，通常比mAP@50低很多。
- 当前值0.25：在严格标准下检测正确率约25%。mAP@50-95通常是mAP@50的一半左右，属正常。

### mAP@50 vs mAP@50-95 的关系
- 差距大（如0.51 vs 0.25）：框的位置不够精确，虽找到了目标但框偏了
- 差距小：定位精准
- 训练初期差距大是正常的，随训练进行差距会缩小

## 2. Precision（精确率）与 Recall（召回率）

这两个指标是**互相对抗**的，提高一个通常意味着降低另一个。

### Precision（精确率，查准率）
- 公式：`TP / (TP + FP)` = 正确检测数 / 总检测数
- 含义：**模型说"这是目标"时，有多大概率真的是目标**。
- 高精确率（>0.9）：模型很保守，只对非常确定的目标才报告 → 漏检多
- 低精确率（<0.3）：模型乱报，大量误检
- 当前值0.70：模型每报10个目标，有7个是正确的，3个是误检。

### Recall（召回率，查全率）
- 公式：`TP / (TP + FN)` = 正确检测数 / 真实目标总数
- 含义：**所有真实目标中，模型找到了多少个**。
- 高召回率（>0.9）：几乎所有目标都找到了 → 但可能有很多误检
- 低召回率（<0.3）：大量目标被漏掉
- 当前值0.47：真实存在的目标中，模型只找到了47%。

### P-R 权衡分析
| 场景 | Precision | Recall | 诊断 |
|------|-----------|--------|------|
| P>>R | 0.70 | 0.47 | **模型偏保守**（当前状态），置信度阈值偏高 |
| R>>P | 0.30 | 0.90 | 模型太激进，大量误检 |
| P≈R | 0.60 | 0.60 | 平衡状态 |

当前P=0.70, R=0.47说明模型倾向于"宁可漏检也不误检"，训练后期Recall会逐渐回升。

## 3. Loss（损失函数）

损失是模型在训练集上的"错误程度"，越低越好。训练目标就是最小化这些损失。

### Box Loss（边界框回归损失）
- 含义：预测框位置与真实框位置的差距。
- 使用CIoU/D-IoU等高级IoU损失计算。
- 当前值1.92：框的位置还有改进空间，持续下降中。
- 降到1.5以下说明定位基本准确。

### Cls Loss（分类损失）
- 含义：预测类别（person/vehicle）与真实类别的差距。
- 使用二元交叉熵（BCE）计算。
- 当前值0.93：模型能较好区分person和vehicle，但仍有混淆。
- 降到0.5以下说明类别区分很清晰。

### DFL Loss（Distribution Focal Loss，分布焦点损失）
- 含义：框边缘概率分布的焦点程度，YOLOv8+新增。
- 作用：让模型对框边界位置的预测更"确定"而非模糊。
- 数值很小（0.003级别），不用太关注绝对值，只要趋势下降即可。

### 损失趋势解读
| 现象 | 含义 |
|------|------|
| 训练损失↓ 验证损失↓ | 正常学习，模型在进步 |
| 训练损失↓ 验证损失↑ | **过拟合**，模型死记训练集，需停止 |
| 训练损失→ 验证损失→ | **平台期**，学习率需调整 |
| 训练损失↑ | 学习率过大或数据有问题 |

## 4. Learning Rate（学习率）

- 含义：模型参数每次更新的步长。
- 当前策略：**Cosine调度**，从0.01开始，先预热3轮（warmup），然后余弦衰减到0.0001。
- lr/pg0：主干网络参数的学习率
- lr/pg1：检测头参数的学习率
- lr/pg2：偏置项的学习率（通常更大）

**学习率与训练阶段：**
- 高（0.01→0.005）：快速学习阶段（前10轮）
- 中（0.005→0.001）：精细调整阶段（10-40轮）
- 低（0.001→0.0001）：微调收敛阶段（40-60+轮）

---

## 5. 训练阶段诊断速查表

| 阶段 | mAP变化 | Loss变化 | P-R状态 | 行动 |
|------|---------|----------|---------|------|
| 初期(1-10轮) | 快速增长 | 快速下降 | 同步上升 | 继续训练 |
| 中期(10-30轮) | 缓慢增长 | 缓慢下降 | P↑R↓或P↓R↑ | 正常调整期 |
| 后期(30-60轮) | 微增或震荡 | 基本稳定 | P和R逐渐平衡 | 监控过拟合 |
| 延伸(60-200轮) | 微小增长 | 稳定 | P≈R | 可提前停止 |

---

## 6. 我们的训练现状诊断

```
Epoch 14 | mAP@50:0.51 | mAP@50-95:0.25 | P:0.70 R:0.47
```

- **处于中期阶段**：快速收敛期已过，进入精细调整
- **P-R缺口0.23**：模型偏保守，后续epoch中Recall应回升至0.55+
- **损失仍在下降**：说明还有学习空间，未到平台期
- **无过拟合**：验证损失与训练损失差距正常
